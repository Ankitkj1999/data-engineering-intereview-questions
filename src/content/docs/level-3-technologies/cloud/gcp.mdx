## GCP: Google Cloud Platform Data Services

**Google Cloud Platform gives data engineers a set of managed services — BigQuery for warehousing, Dataflow and Dataproc for processing, Pub/Sub and Composer for orchestration — built on top of IAM, VPC networking, and reliability primitives that span all of them.**

## Key Concepts

GCP's data stack splits cleanly by workload shape. **BigQuery** is the serverless, SQL-first data warehouse for large-scale analytics; **Cloud Storage** is the object store most pipelines land raw data in first; and **Cloud SQL**, **Cloud Spanner**, **Cloud Datastore**, and **Memorystore** cover relational, globally-distributed relational, NoSQL document, and in-memory workloads respectively. For movement and transformation, **Cloud Dataflow** runs serverless batch/stream pipelines, **Cloud Dataproc** runs managed Spark/Hadoop clusters, **Cloud Pub/Sub** handles asynchronous messaging, and **Cloud Composer** (managed Apache Airflow) orchestrates the dependencies between all of it. Across every service, **Cloud IAM** controls who can do what via fine-grained policies, **VPC** provides network isolation with subnets and firewall rules, and a family of networking services (**Cloud Load Balancing**, **Cloud CDN**, **Cloud DNS**, **Cloud NAT**, **Cloud VPN**) handle traffic distribution and connectivity. Reliability is a shared responsibility across regional/multi-regional deployment, automated backups, and data replication — the same primitives show up whether you're talking about high availability, disaster recovery, or redundancy, because they're solving the same underlying problem from different angles.

Google Cloud Platform is a suite of cloud computing services provided by Google, offering a wide range of infrastructure and platform services for building, deploying, and managing applications and data.

<QuestionCard id="gcp-core-02" number={2} question="What are the key components of GCP?" difficulty="easy" topic="Fundamentals">
GCP comprises various services across compute, storage, networking, databases, big data, machine learning, and management/monitoring tooling.
</QuestionCard>

<QuestionCard id="gcp-core-03" number={3} question="What are the advantages of using GCP for data engineering?" difficulty="easy" topic="Fundamentals">
GCP provides scalability, flexibility, high-performance computing, a range of data storage options, managed services that reduce operational overhead, strong security, and tight integration with other Google services.
</QuestionCard>

<QuestionCard id="gcp-core-04" number={4} question="How does GCP store data?" difficulty="medium" topic="Storage">
GCP offers a range of storage options depending on the shape of your data: Cloud Storage for object storage, Cloud SQL for managed relational databases, Bigtable for a NoSQL wide-column store, and Cloud Firestore for document-based NoSQL data.
</QuestionCard>

<QuestionCard id="gcp-core-05" number={5} question="What is Google BigQuery?" difficulty="easy" topic="Analytics">
BigQuery is a fully managed, serverless data warehouse provided by GCP. It lets you analyze massive datasets using SQL queries with high speed and scalability.
</QuestionCard>

<QuestionCard id="gcp-core-06" number={6} question="Explain the role of Cloud Storage in GCP." difficulty="easy" topic="Storage">
Cloud Storage is an object storage service provided by GCP. It offers scalable, durable, and highly available storage for objects of any size, and is commonly used for storing files, backups, and serving static content.
</QuestionCard>

<QuestionCard id="gcp-core-07" number={7} question="What is Cloud Spanner in GCP?" difficulty="medium" topic="Databases">
Cloud Spanner is a horizontally scalable, globally distributed relational database service provided by GCP. It combines the transactional consistency of relational databases with the horizontal scalability more typical of NoSQL systems.
</QuestionCard>

<QuestionCard id="gcp-core-08" number={8} question="What is Cloud SQL and how is it used in GCP?" difficulty="easy" topic="Databases">
Cloud SQL is a managed database service in GCP that provides MySQL, PostgreSQL, and SQL Server databases. It simplifies database management tasks, offers high availability, and scales seamlessly.
</QuestionCard>

<QuestionCard id="gcp-core-09" number={9} question="Explain the use of Cloud Datastore in GCP." difficulty="medium" topic="Databases">
Cloud Datastore is a NoSQL document database provided by GCP. It's schemaless, scales automatically, and is well suited to applications that require low-latency reads and writes.
</QuestionCard>

<QuestionCard id="gcp-core-10" number={10} question="What is Memorystore in GCP?" difficulty="medium" topic="Caching">
Cloud Memorystore is a fully managed in-memory data store service. Its Redis offering provides high-performance, scalable Redis instances for caching and fast data access, without you having to manage the underlying infrastructure.
</QuestionCard>

<QuestionCard id="gcp-core-11" number={11} question="What is the purpose of Cloud SQL Proxy in GCP?" difficulty="medium" topic="Databases">
Cloud SQL Proxy is a secure client-side proxy for connecting to Cloud SQL instances from external applications or local development environments, without exposing the database directly to the internet.
</QuestionCard>

<QuestionCard id="gcp-core-12" number={12} question="How does GCP handle data archiving and long-term storage?" difficulty="medium" topic="Storage">
GCP offers services like Cloud Storage Coldline and Archive for long-term data storage, providing cost-effective options for archiving data while maintaining high durability and availability.
</QuestionCard>

### Data Processing, Ingestion & Machine Learning

<QuestionList topic="GCP Data Processing" storagePrefix="gcp-processing">

You can use services like Cloud Storage, Cloud Pub/Sub, Data Transfer Service, or third-party tools to import data into GCP for analysis.

<QuestionCard id="gcp-processing-02" number={14} question="Explain the concept and use of Cloud Dataflow in GCP." difficulty="medium" topic="Processing">
Cloud Dataflow is a serverless data processing service for building, deploying, and executing data pipelines that transform and process data in parallel. It supports both batch and stream processing, and integrates with other GCP services like BigQuery.
</QuestionCard>

<QuestionCard id="gcp-processing-03" number={15} question="What is Cloud Dataproc?" difficulty="medium" topic="Processing">
Cloud Dataproc is a fully managed service in GCP for running Apache Spark and Apache Hadoop clusters. It provides a scalable and cost-effective way to process big data workloads.
</QuestionCard>

<QuestionCard id="gcp-processing-04" number={16} question="What is Cloud Pub/Sub?" difficulty="easy" topic="Messaging">
Cloud Pub/Sub is a messaging service in GCP that enables asynchronous communication between independent applications. It lets you build scalable, event-driven architectures.
</QuestionCard>

<QuestionCard id="gcp-processing-05" number={17} question="What is Cloud Composer in GCP and what is it used for?" difficulty="medium" topic="Orchestration">
Cloud Composer is a fully managed workflow orchestration service based on Apache Airflow. It lets you create, schedule, and monitor workflows composed of tasks — managing dependencies between them — for data processing and ETL pipelines.
</QuestionCard>

<QuestionCard id="gcp-processing-06" number={18} question="What is the purpose of Cloud Machine Learning Engine in GCP?" difficulty="medium" topic="Machine Learning">
Cloud Machine Learning Engine is a managed service for building, training, deploying, and serving machine learning models at scale. It supports popular frameworks like TensorFlow and scikit-learn, and integrates with other GCP services like BigQuery and Cloud Storage.
</QuestionCard>

<QuestionCard id="gcp-processing-07" number={19} question="Explain the use of Cloud AutoML in GCP." difficulty="medium" topic="Machine Learning">
Cloud AutoML is a suite of machine learning products in GCP that lets you train custom machine learning models with minimal coding. It supports vision, natural language, and tabular data use cases.
</QuestionCard>

<QuestionCard id="gcp-processing-08" number={20} question="How can you optimize data ingestion in GCP?" difficulty="medium" topic="Ingestion">
GCP provides services like Cloud Pub/Sub and Cloud Dataflow for efficient, scalable data ingestion. You can further optimize ingestion using partitioning, batching, and compression techniques.
</QuestionCard>

<QuestionCard id="gcp-processing-09" number={21} question="Explain the use of Cloud Datalab in GCP." difficulty="easy" topic="Analytics Tools">
Cloud Datalab is an interactive data exploration and visualization tool in GCP. It provides a Jupyter notebook interface for analyzing and visualizing data using Python, SQL, and BigQuery.
</QuestionCard>

<QuestionCard id="gcp-processing-10" number={22} question="How can you optimize data processing in GCP?" difficulty="medium" topic="Processing">
GCP offers optimization techniques like data partitioning, distributed processing, caching, and choosing the appropriate storage and processing services for the workload, to achieve efficient and scalable data processing.
</QuestionCard>

<QuestionCard id="gcp-processing-11" number={23} question="Explain the use of Cloud IoT Core in GCP." difficulty="medium" topic="IoT">

## Common Interview Scenarios

- **"Design a pipeline that ingests IoT sensor data and makes it queryable in near real time."** Talk through Pub/Sub as the ingestion buffer, Dataflow for streaming transformation, and BigQuery as the query target, with Composer coordinating anything batch-oriented alongside it.
- **"A team needs to give a third-party analytics vendor read-only access to one BigQuery dataset, and nothing else."** Walk through IAM's fine-grained, resource-level policies versus project-wide roles, and why you'd scope the grant to the dataset rather than the project.
- **"Your workload needs to survive a full regional outage with minimal data loss."** Discuss multi-regional deployment, automated backups, and cross-region replication — and how the same primitives underpin GCP's answers to high availability, disaster recovery, and backup/redundancy questions.
- **"How would you keep a growing GCP bill under control without slowing teams down?"** Cover Cloud Billing budgets and alerts, right-sizing with Cloud Monitoring data, and storage-class choices like Coldline/Archive for infrequently accessed data.
